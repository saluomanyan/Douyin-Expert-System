# Memory Standard

> Hermes 专家长期记忆的设计规范

---

## 一、Memory 是什么

Memory 不是"提示词"，不是"角色设定"，不是"运营技巧合集"。

Memory 是**经过人工审核、多源验证、知识蒸馏的专家认知资产**。

它应该像操盘手大脑里默认存在的知识：不需要搜索、不需要推理、天然就知道。

---

## 二、Memory 分级体系

| 等级 | 名称 | 内容 | 更新频率 | 锁定期 |
|------|------|------|----------|--------|
| **L1** | 世界观 | 为什么做内容、为什么做人设、为什么做 IP、创作哲学 | 几乎不改 | 永久 |
| **L2** | 战略 | 为什么账号增长、用户为什么关注、停留率为什么是一切 | 数年更新 | 12 个月 |
| **L3** | 战术 | 昵称、简介、封面、标题、黄金三秒、剪辑、脚本 | 按需迭代 | 6 个月 |
| **L4** | 平台规则 | 当前算法机制、流量池规则、平台政策 | 变化最快 | 3 个月 |

> **锁定期**：Memory 创建后 N 个月内不得修改（除非发现事实性错误）。
> 这是为了防止反复横跳、认知漂移。

---

## 三、Memory 收录标准（6 条铁律）

### ✅ 必须满足（全部 6 条）

1. **操作锚点**：能指导至少一个具体动作
2. **解释"为什么"**：揭示底层机制，而非仅说"怎么做"
3. **多源验证**：至少 3 个独立来源交叉验证
4. **跨赛道复用**：优先底层规律，不收录单一赛道技巧
5. **与抖音生态强相关**：不是泛互联网运营理论
6. **不违背创作哲学**：长期 IP > 短期流量

### ❌ 禁止收录（示例）

| 类型 | 示例 | 问题 |
|------|------|------|
| 废话 | "做好定位"、"坚持更新" | 无操作锚点 |
| 个人经验 | "我这样做就爆了" | 未多源验证 |
| 平台规则复述 | "抖音推荐靠算法" | 没有解释为什么 |
| 热点技巧 | "2026 年流量扶持政策" | 不可跨时间复用 |
| 创作者哲学冲突 | "标题一定要制造焦虑" | 损害长期 IP |

## 四、Memory 文件格式

### 文件名

```
{等级目录}/{M编号}_{标题}.md

示例：
L3_Tactics/M-015_封面设计.md
L2_Strategy/M-004_用户行为动机.md
```

### 文件内容模板

```markdown
---
memory_id: M-XXX
title: {标题}
level: {L1 | L2 | L3 | L4}
version: 1.0
credibility: {★~★★★★★}
sources:
  - {来源1：名称 + URL/出处}
  - {来源2}
  - {来源3}
applicable:
  - {适用场景1}
  - {适用场景2}
not_applicable:
  - {不适用场景1}
tags: [tag1, tag2, tag3]
created: YYYY-MM-DD
last_updated: YYYY-MM-DD
reviewed_by: {审核人}
locked_until: YYYY-MM-DD
---

# {标题}

## 核心认知

{一句话表达最核心的认知。这句话应该像 CPU 指令一样直接。}

## 为什么

{解释底层机制。不是"怎么做"，而是"为什么这样做有效"。}

## 怎么做

{具体操作步骤，每步必须具有操作锚点。}

## 适用边界

{什么时候这个原则成立？什么时候不成立？}

## 常见误区

{大多数人会犯的错。}

## 证据支撑

{多源验证的简要说明。}

## 相关 Memory

- [M-XXX](../L{level}/{文件名}.md) — 相关说明
```

## 五、可信度评级

| 星级 | 含义 | 条件 |
|------|------|------|
| ★★★★★ | 行业共识 | ≥5 个顶级操盘手一致认同 + 官方资料支持 + A/B 测试验证 |
| ★★★★☆ | 高度可信 | ≥3 个操盘手认同 + 官方课程提及 |
| ★★★☆☆ | 可信 | ≥3 个来源交叉验证 |
| ★★☆☆☆ | 待验证 | 2 个来源，需要更多验证 |
| ★☆☆☆☆ | 假设 | 单一来源，仅供参考 |

> Memory 最低要求 **★★★☆☆**。
> ★★☆☆☆ 及以下只留在 `knowledge/`，不进入 `memory/`。

## 六、审核流程

```
提议新 Memory
    ↓
检查是否满足 6 条铁律 ← 不满足 → 拒绝或留在 knowledge/
    ↓
多源验证（≥3 来源）  ← 不足 → 标记为待验证
    ↓
人工审核              ← 不通过 → 修改后重新提交
    ↓
分配等级 + 编号
    ↓
进入 memory/ 对应目录
    ↓
记录到 config/memory-index.md
```
