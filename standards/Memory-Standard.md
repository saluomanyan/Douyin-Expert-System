# Memory Standard · 记忆规范

> 定义 Hermes-Douyin-Expert-System 中每条 Memory 的标准格式。

---

## Memory 编号规则

```
M<4位数字>

示例：M0001、M0012、M0103
```

编号由项目维护者统一分配。一个编号只对应一条 Memory，永不重用。

| 层级 | 名称 | 定义 | 示例 |
|------|------|------|------|

## Memory 条目标准字段

每条 Memory 的 `.md` 文件必须包含以下字段：

```yaml
---
id: M0001                           # 唯一编号
title: "创作者哲学"                   # 中文标题
level: L1                           # L1/L2/L3/L4
category: "Worldview"               # 分类标签
tags: [创作者, 底层逻辑, 认知]        # 标签（3-8个）
confidence: high                    # high / medium / low
sources:                            # 来源列表（3+）
  - name: "来源名称"
    type: "book|course|article|interview|report"
    url: "https://..."
    note: "参考页码/章节"
last_updated: 2026-06-27            # 最后更新日期
version: 1                          # 版本号
---
```

## 内容格式要求

### 必须包含

1. **一句话摘要** — 用一句话说清这条认知的核心
2. **为什么重要** — 这条认知为什么是基石而非技巧
3. **核心认知** — 3-7 条精炼的认知陈述（每条 1-3 句）
4. **常见误区** — 大多数人在这件事上错在哪
5. **如何应用** — 不是教程，但给出推演方向

### 禁止包含

- ❌ 操作步骤（那是 Checklist/Skill 的事）
- ❌ 平台规则复述（那是 Knowledge 的事）
- ❌ 经验故事（不具可复用性）
- ❌ "建议"、"可以尝试"等模糊用词
- ❌ 超过 500 字的单条 Memory

### 语言风格

- 断言式语句：直接给出结论，不用"可能"、"大概"
- 解释"为什么"而非"怎么做"
- 每条认知能独立成立，也能和其他记忆互相关联
- 像操盘手脑中的默认信念，而非教程段落

## 可信度标注

| 级别 | 含义 | 条件 |
|------|------|------|
| **high** | 业界共识，多源交叉验证 | 5+ 独立来源，且无相反意见 |
| **medium** | 有一定依据，但来源有限 | 3-4 个来源，或存在轻微分歧 |
| **low** | 个人经验/推测 | 不足 3 个来源，标记为待验证 |

---

## 交叉验证要求

每条 Memory 进入 `confidence: high` 之前，必须满足：

1. 至少 **3 个独立来源** 支持核心主张
2. 来源类型至少覆盖 **2 种不同类型**（如：课程 + 行业报告 + 操盘手访谈）
3. 未发现明确的相反观点
