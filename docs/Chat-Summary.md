## 一、核心目标

构建一个 **「抖音专家认知系统 (Douyin Expert Operating System)」**，作为 Hermes Agent 的私有专家记忆库。

**不是：**
- 一个 Prompt
- 一个 AI 角色扮演
- 一个 Skill 合集
- 依赖 LLM 自带知识的问答

**而是：**
- 一套经过人工审核、多源验证、知识蒸馏的**专家认知资产**
- 完全独立于 LLM，换模型不影响知识质量
- 能够被 Hermes Skills 调用的长期 Memory

## 二、用户画像与账号定位

### 创作者的账号方向

| 主线 | 内容 | 重点 |
|------|------|------|
| 自然探索 | 山、湖、森林、海、日落、四季 | 视觉 |
| 人文探索 | 城市、街道、建筑、烟火气 | 故事 |
| 精神探索 | 旅行思考、孤独、成长、自由 | 思想 |

### 核心理念（创作哲学）

- **大道至简**：宁愿少说，不要废话，让画面自己说话
- **真实比完美重要**：不过度调色、不过度演、不过度制造情绪
- **尊重风景**：不是利用风景制造流量，而是帮助观众看见风景
- **视频不是攻略**：视频是一种体验，攻略只是辅助
- **长期 IP > 短期流量**：任何运营建议不得损害长期 IP

### 个人 IP 策略

- 露脸不是为了"增加信任"，而是让观众记住：**"是这个人带我去看世界"**
- 风景是载体，创作者是连接观众与世界的那个人

## 三、四层知识架构

```
repository/     ← 原始资料（课程、文章、案例、官方文档，永不直接喂给 Agent）
    ↓  人工 + AI 蒸馏
knowledge/      ← 提炼后的知识（经过整理和摘要）
    ↓  进一步蒸馏 + 多源交叉验证
memory/         ← 专家长期记忆（最终进入 Hermes 的认知资产）
    ↓  被调用
skills/         ← 执行能力（调用 Memory 完成实际工作）
```

---

## 四、Memory 分层体系

| 等级 | 名称 | 内容 | 更新频率 |
|------|------|------|----------|
| L1 | 世界观 | 为什么做内容、为什么做人设、为什么做 IP | 几乎不改 |
| L2 | 战略 | 为什么账号增长、为什么用户关注、为什么停留率 | 数年更新 |
| L3 | 战术 | 昵称、简介、封面、剪辑、脚本 | 按需迭代 |
| L4 | 平台规则 | 当前算法、流量机制、平台政策 | 变化最快 |

---

## 五、Memory 收录标准（6 条铁律）

1. **必须具有明确操作锚点**（能指导具体动作，不能是"做好定位"这种废话）
2. **必须解释"为什么"**（不仅告诉做什么，还揭示底层机制）
3. **尽量有多来源验证**（至少 3 个独立来源交叉验证）
4. **能够跨赛道复用**（优先收录底层规律，不收录短期技巧）
5. **与抖音生态强相关**（不是泛互联网运营理论）
6. **不违背创作哲学**（长期 IP 优先于短期流量）

### 禁止收录的类型

- "内容要有价值""坚持更新""做好定位"等无操作锚点的废话
- 单纯的经验故事（不能迁移）
- 平台规则复述（没有解释"为什么"）
- 碎片技巧（不能融入完整决策体系）

### 每条 Memory 的元数据字段

```
编号、标题、内容、可信度（★~★★★★★）、
来源列表、适用范围、不适用范围、更新时间
```

## 七、六阶段实施路线

| 阶段 | 内容 | 产出 |
|------|------|------|
| Phase 1 | 建立工程骨架 | README、标准、模板、配置（**不写任何知识**） |
| Phase 2 | 建设 Memory | 按 L1→L4 逐级填充专家长期记忆 |
| Phase 3 | 建设 Framework | 建立所有分析框架 |
| Phase 4 | 建设 Checklist | 所有操作流程形成检查清单 |
| Phase 5 | 建设 Skills | Skills 优先调用 Memory→Framework→Checklist，LLM 仅组织语言 |
| Phase 6 | 接入 Agent | Hermes/LangGraph/ClaudeCode 适配器 |

## 八、Agent 交互规范（"严苛合伙人"模式）

作为专家，Agent 必须遵循：

1. **第一职责：否定** —— 不盲目迎合，先质疑可行性、竞争、差异化、商业闭环
2. **第二职责：拆逻辑** —— 用 Decision Tree 推演（用户为什么看？为什么现在看？为什么非看你？……）
3. **第三职责：给结论** —— 必须有明确判断：「可以做」「不建议」「可以做但必须改」
4. **第四职责：给替代方案** —— 方向不行时，明确说"不要做 A，应该做 B，因为……"
