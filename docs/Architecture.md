# Architecture

> Hermes-Douyin-Expert-System 系统架构

---

## 分层架构

本工程采用严格的四层知识蒸馏架构：

```
┌─────────────────────────────────────────────────┐
│                  Agent Layer                     │
│  Hermes / LangGraph / ClaudeCode / OpenManus     │
│         (通过 adapter/ 目录适配)                  │
└──────────────────────┬──────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────┐
│                 Skills Layer                      │
│  调用 Memory → Framework → Checklist 完成工作     │
│  LLM 仅负责组织语言，不做决策                      │
└──────────────────────┬──────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────┐
│                Memory Layer                       │
│  L1 世界观 │ L2 战略 │ L3 战术 │ L4 平台规则       │
│  (整个系统最有价值、最稀缺的资产)                   │
└──────────────────────┬──────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────┐
│              Knowledge Layer                      │
│  经过整理和摘要的知识                             │
└──────────────────────┬──────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────┐
│              Repository Layer                     │
│  原始资料（永不直接喂给 Agent）                    │
└─────────────────────────────────────────────────┘
```

---

## 数据流

```
用户输入
    ↓
Skill 判断任务类型
    ↓
读取相关 Memory（获取专家认知）
    ↓
应用 Framework（结构化分析）
    ↓
对照 Checklist（逐项审核）
    ↓
LLM 组织语言输出
```

**注意**：LLM 在整个流程中仅负责最后一步"组织语言"，不做推理决策。真正的决策依据全部来自 Memory。

---

## 关键设计决策

### 1. 知识独立于模型

- Memory 独立于任何 LLM（DeepSeek / Qwen / Claude / GPT）
- 换模型不影响知识质量
- 真正的知识永远在 Memory 里，不在 LLM 的训练参数里

### 2. 先知识工程，后 Agent 工程

- 第一版不绑定任何 Agent 框架
- 通过 `adapter/` 目录在 Phase 6 接入 Hermes 等
- 知识工程本身保持框架无关

### 3. 以 Git Commit 为推进单位

- 不按"模块"推进，按"可交付物"推进
- 每次 Commit 对应一个可审核、可回滚的知识沉淀

## 目录职责

| 目录 | 职责 | 说明 |
|------|------|------|
| `memory/` | 专家长期记忆 | **核心资产**，经过多源验证和人工审核 |
| `skills/` | 执行能力 | 调用 Memory 完成具体任务 |
| `framework/` | 分析框架 | 结构化分析方法论 |
| `checklist/` | 检查清单 | 可执行的操作审核标准 |
| `repository/` | 原始资料 | 未加工素材，不可直接喂给 Agent |
| `knowledge/` | 提炼知识 | 已整理摘要，但尚未达到 Memory 标准 |
| `standards/` | 规范 | 各类文件的设计标准 |
| `templates/` | 模板 | 新建文件时的标准模板 |
| `adapter/` | 适配器 | 将知识工程接入不同 Agent 框架 |
| `config/` | 索引 | 跨文件索引和标签 |
| `prompts/` | AI Prompt | 辅助 AI 进行知识提取的 Prompt |
| `docs/` | 文档 | 项目说明文档 |
| `archive/` | 存档 | 废弃/草稿/实验内容 |
