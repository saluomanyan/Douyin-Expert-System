# Architecture v0.2

> Hermes-Douyin-Expert-System 系统架构

---

## 分层架构

本工程采用四层知识粒度 + 四层蒸馏架构：

### 知识粒度（v0.2 新增 MA 层）

```
┌─────────────────────────────────────────────────┐
│                  Agent Layer                     │
│  Hermes / LangGraph / ClaudeCode / OpenManus     │
│         (通过 adapter/ 目录适配)                  │
└──────────────────────┬──────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────┐
│                 Skills Layer (S)                  │
│  调用 MA → M → F → Checklist 完成工作             │
│  LLM 仅负责组织语言，不做决策                      │
└──────────────────────┬──────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────┐
│               Framework Layer (F)                 │
│  结构化分析方法论，调用多个 M + MA                  │
└──────────────────────┬──────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────┐
│                Memory Layer (M)                   │
│  L1 世界观 │ L2 战略 │ L3 战术 │ L4 平台规则       │
│  每篇 M 由多个 MA（记忆原子）聚合而成               │
│  (整个系统最有价值、最稀缺的资产)                   │
└──────────────────────┬──────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────┐
│             Memory Atom Layer (MA)                │
│  不可再拆的原子认知（一句话原则）                    │
│  知识的最小可复用单元                               │
└──────────────────────┬──────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────┐
│              Knowledge Layer                      │
│  经过整理和摘要的知识                             │
└──────────────────────┬──────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────┐
│              Repository Layer                     │
│  原始资料（永不直接喂给 Agent）                    │
└─────────────────────────────────────────────────┘
```

---

## 知识流向

```
Repository（原始资料）
    ↓ 阅读 + 提炼
Knowledge（摘要知识）
    ↓ 蒸馏 + 多源验证
MA（记忆原子）──→ M（专家记忆）
    ↓                  ↓
F（分析框架）+ Checklist（检查清单）
    ↓
S（执行能力）
    ↓
Agent（Hermes 等）
```

---

## 数据流（Agent 运行时）

```
用户输入
    ↓
Skill 判断任务类型
    ↓
读取 MA（获取原子认知）
    ↓
读取 M（获取完整专家判断）
    ↓
应用 F（结构化分析）
    ↓
对照 Checklist（逐项审核）
    ↓
LLM 组织语言输出
```

**注意**：LLM 在整个流程中仅负责最后一步"组织语言"，不做推理决策。真正的决策依据全部来自 MA + M。

---

## 关键设计决策

### 1. 知识独立于模型

- Memory 独立于任何 LLM
- 换模型不影响知识质量
- 真正的知识永远在 Memory 里

### 2. 先知识工程，后 Agent 工程

- 通过 `adapter/` 目录在 Phase 6 接入
- 知识工程本身保持框架无关

### 3. 原子化（v0.2）

- MA 是最小可复用单元
- 多个 M 可以共享同一组 MA
- 修改一个 MA，所有引用它的 M 自动受益

## 目录职责

| 目录 | 职责 | 说明 |
|------|------|------|
| `memory/atoms/` | 记忆原子 (MA) | 不可再拆的一句话认知 |
| `memory/L1-L4/` | 专家记忆 (M) | 由 MA 聚合的完整论述 |
| `skills/` | 执行能力 (S) | 调用 MA→M→F→Checklist |
| `framework/` | 分析框架 (F) | 结构化方法论 |
| `checklist/` | 检查清单 | 可执行的操作审核标准 |
| `repository/` | 原始资料 | 未加工素材 |
| `knowledge/` | 提炼知识 | 已整理摘要 |
| `standards/` | 规范 | 设计标准 |
| `templates/` | 模板 | 文件模板 |
| `adapter/` | 适配器 | Agent 框架接入 |
| `config/` | 索引 | 跨文件索引 |
