# 抖音专家系统 — 聊天记录提炼

> 原始对话来源：`和chatGpt的聊天记录.md`（3387行）
> 提炼日期：2026-06-27

---

## 一、核心目标

构建一个运行在 **Hermes Agent** 上的 **抖音运营专家认知系统 (Knowledge System)**。

**不是**：一个 Prompt、一个角色扮演、一个普通 Skill。
**而是**：一套自带高质量长期记忆 + 方法论 + 分析框架的独立知识工程。

---

## 二、三层架构

```
抖音专家 Agent
│
├── Skills（执行能力）      ← 会做什么：分析选题、生成脚本、账号诊断……
│
├── Memory（长期认知）      ← 默认知道什么：爆款底层逻辑、算法机制、用户心理……
│
└── Methodology（方法论）   ← 决策 SOP/决策树：拿到选题后的推演流程
```

**核心洞察**：Memory 是整个系统最有价值的部分。它独立于 LLM，换任何模型（Claude/Qwen/DeepSeek/GPT）都不影响认知质量。

---

## 三、Memory（专家记忆）的特征要求

每条记忆必须满足：
- **不是经验故事** → 是可复用的原则（跨赛道迁移）
- **不是平台规则复述** → 是底层机制解释（知道"为什么"）
- **不是碎片技巧** → 能组成完整决策体系（知识间可相互引用推演）
- **不是教程** → 是操盘手脑内的 CPU 指令集

示例：
> ❌ "写一个吸引人的标题"
> ✅ "标题不是介绍视频，而是制造点击理由"

---

## 四、四层知识流向（V0.2 演进目标）

```
原始资料 (repository/)    ← 未经加工的课程、文章、案例
       ↓
知识提炼 (knowledge/)     ← 整理和摘要后的知识
       ↓
专家记忆 (memory/)       ← 最终进入 Hermes 的长期认知
       ↓
执行能力 (skills/)       ← 调用 Memory 完成实际工作
```

---

## 五、Agent 工作方式（合伙人模式）

Agent 的四个职责：

| 职责 | 行为 | 示例 |
|------|------|------|
| **1. 否定** | 第一反应不是认同，而是质疑 | "竞争是否过大？差异化在哪？商业闭环？" |
| **2. 拆逻辑** | 用决策树推演 | "解决谁的问题？为什么现在需要？为什么来抖音？" |
| **3. 给结论** | 必须有明确判断 | "不建议"/"可以做"/"可以做但必须改" |
| **4. 给替代方案** | 不行就说为什么，并给出 B 方案 | "不要做A，应该做B，因为……" |

## 七、设计原则

1. **先知识工程，再 Agent 工程** — 不绑定 Hermes，任何 Agent 框架都能加载
2. **以 GitHub Commit 为推进单位** — 每次对话对应一次可追踪、可回滚的知识沉淀
3. **知识来源交叉验证** — 至少 10 个来源验证同一认知才收录
4. **人工二次审核** — AI 负责搜集提炼，人负责最终筛选入库
5. **知识独立于 LLM** — 换模型不影响认知质量
