# CONTRIBUTING

> 如何为本项目贡献知识

---

## 贡献原则

### 核心原则：宁缺毋滥

本项目不是"抖音运营技巧合集"。每条进入 Memory 的知识必须通过严格的**6 条铁律**审核（详见 [Memory-Standard.md](standards/Memory-Standard.md)）。

### 禁止提交的类型

- "做好定位""坚持更新"等无操作锚点的废话
- 未经验证的个人经验
- 仅搬运他人内容而未做多源验证
- 与创作者哲学冲突的技巧（如制造焦虑、标题党等）

---

## 贡献流程

1. **Fork** 本仓库
2. 在 `repository/` 目录下添加原始资料
3. 在 `knowledge/` 目录下提交提炼后的知识摘要
4. 在 `memory/` 目录下按规范提交 Memory 文件
5. 发起 **Pull Request**，附上：
   - 信息来源（至少 3 个独立来源）
   - 为什么值得进入 Memory（按 6 条铁律逐条说明）

---

## Memory 文件格式

参见 [templates/Memory.md](templates/Memory.md)

---

## 审核标准

PR 将按以下标准审核：

1. 是否具有明确操作锚点？
2. 是否解释了"为什么"？
3. 是否有 ≥3 个独立来源验证？
4. 是否能跨赛道复用？
5. 是否与抖音生态强相关？
6. 是否违背创作者哲学？
